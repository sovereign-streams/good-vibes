1 Enrichment Envelope

This example shows a fully enriched record for a YouTube video about building a surfboard from scratch. This is what the enrichment pipeline produces after processing raw YouTube metadata through the LLM tagging pass.

Enrichment Envelope — Surfboard Video 
{
  "sep_version": "0.1.0",
  "item_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "source": {
    "platform": "youtube",
    "origin_url": "https://youtube.com/watch?v=xK3fRmPzQuo",
    "origin_id": "xK3fRmPzQuo",
    "content_type": "video",
    "duration_seconds": 342
  },
  "meta": {
    "title": "How to Build a Surfboard from Scratch",
    "creator": "Shaping Bay",
    "published": "2026-01-15T00:00:00Z",
    "original_tags": ["surfboard", "woodworking", "craft"],
    "language": "en",
    "thumbnail_url": "https://i.ytimg.com/vi/xK3fRmPzQuo/maxresdefault.jpg"
  },
  "enrichment": {
    "schema_version": "0.1.0",
    "enriched_at": "2026-02-28T12:00:00Z",
    "categories": [
      { "id": "craft", "confidence": 0.92 },
      { "id": "skill_building", "confidence": 0.85 },
      { "id": "relaxation", "confidence": 0.71 }
    ],
    "emotional_tone": {
      "primary": "calm",
      "secondary": "focused",
      "rage_bait": false,
      "humiliation": false,
      "shock_content": false,
      "inflammatory": false
    },
    "energy_level": 0.35,
    "cognitive_load": 0.45,
    "motivation_score": 0.40,
    "humor_score": 0.10,
    "skill_transfer_score": 0.80,
    "session_fit": {
      "good_opener": false,
      "good_builder": true,
      "good_peak": false,
      "good_closer": true
    }
  },
  "provider": {
    "id": "good-vibes-main",
    "guardrail_pass": true,
    "guardrail_version": "0.1.0"
  }
}

Walkthrough

source — Points to the original YouTube video. The SEP envelope never hosts the content — it references the origin. The consumer's shell lazy-loads the actual video only when the user is ready to watch it. This keeps the protocol lightweight: you're exchanging metadata, not video blobs.
meta — The raw metadata from YouTube: title, creator, publish date, original tags, language. This is preserved exactly as it came from the source API. The enrichment layer builds on top of this — it doesn't replace it.
enrichment.categories — The LLM tagger identified this as primarily a craft video (0.92 confidence), with strong skill_building (0.85) and moderate relaxation (0.71) signals. A content item can match multiple categories. The PAE uses these confidence scores to weight the item against the user's category preferences.
enrichment.emotional_tone — The primary tone is calm with a secondary of focused. All safety flags are false — no rage-bait, humiliation, shock, or inflammatory content detected. This is a clean, educational video with a meditative quality.
Scores — Low energy (0.35) and moderate cognitive load (0.45) — this is a watch-and-absorb video, not a jump-up-and-act one. High skill transfer (0.80) means you'll actually learn something. Low humor (0.10) — it's instructional, not comedy.
session_fit — Good as a builder (mid-session, focused learning) and good as a closer (calming, grounding). Not a good opener (too focused to start cold) or peak (not enough energy). The PAE will slot this into the appropriate position in a session arc.
provider — This item passed Good Vibes guardrails (version 0.1.0). It's safe to include in Good Vibes curated streams. The guardrail check is a binary pass/fail — items that fail are still stored but excluded from Good Vibes responses.

2 Consumer Intent — Morning Warrior Session

This is what a consumer's PAE sends to a provider when requesting a "Morning Warrior" session — a high-energy morning routine focused on fitness and motivation, with humor to warm up and music to close.

Consumer Intent — Morning Warrior 
{
  "sep_version": "0.1.0",
  "consumer_id": "anonymous-pae-7f3a",
  "intent": {
    "session_type": "composed",
    "target_duration_minutes": 15,
    "weights": {
      "fitness": 0.30,
      "motivation": 0.25,
      "humor": 0.15,
      "skill_building": 0.10,
      "music": 0.10,
      "stoicism": 0.10
    },
    "filters": {
      "exclude_rage_bait": true,
      "exclude_humiliation": true,
      "exclude_shock_content": true,
      "min_energy_level": 0.3,
      "max_cognitive_load": 0.6,
      "language": ["en"]
    },
    "context": {
      "time_of_day": "morning",
      "session_number_today": 1,
      "state_token": null
    }
  },
  "disclosure_level": "minimal",
  "telemetry_opt_in": false
}

Walkthrough

consumer_id — The consumer identifies as anonymous-pae-7f3a. Combined with disclosure_level: "minimal", the provider gets no behavioral history, no session history, nothing beyond this intent payload. Maximum privacy.
session_type: "composed" — The consumer is requesting a composed session — not a raw list of items, but a payload that can be assembled into a session arc with emotional pacing. The provider will include a suggested_arc in its response.
weights — This is the core of algorithmic sovereignty. The user wants 30% fitness, 25% motivation, 15% humor. These aren't rigid quotas — the provider uses them as preference signals. In a 15-minute session (roughly 8 short-form items), the distribution might be: 2-3 fitness, 2 motivation, 1 humor, 1 skill/stoicism, 1 music.
filters — Hard exclusions: no rage-bait, no humiliation, no shock content. Plus: minimum energy level of 0.3 (it's a morning warrior session — nothing too sleepy) and max cognitive load of 0.6 (morning means moderate focus, not heavy study material). English only.
context — Morning, first session of the day, no prior state token. This tells the provider it's a fresh session — no continuity from a previous exchange. If the user had a state token from a prior interaction, the provider could use it for smarter follow-up without building a persistent profile.
telemetry_opt_in: false — The consumer isn't sharing engagement data after this session. The provider gets the intent, sends the payload, and the exchange ends. No feedback loop. The consumer retains full privacy over their viewing behavior.

Items follow the arc — Item 1 (comedy, good_opener) warms the user up. Items 2-4 (fitness, skill, stoicism) build engagement. Item 5 (Goggins motivation, energy 0.95) is the peak. Items 6-7 (stretch + music) bring the user down to a grounded exit state. The whole session is roughly 15 minutes of designed emotional pacing.

capabilities — Good Vibes supports stateful sessions and telemetry exchange but not full index browsing. If the consumer's PAE wanted Model B (full index browse), it would need a different provider. These capabilities are declared upfront in the provider manifest.

5 Full Exchange Flow

Here's the complete end-to-end flow of a SEP exchange, from enrichment through session experience.

1

Content Ingestion & Enrichment

YouTube API returns raw metadata for 1,000 videos. The enrichment pipeline processes each one: fetches transcripts, runs LLM tagging, checks guardrails, validates against schema. 850 items pass guardrails and are stored as SEP-compliant enrichment envelopes in the provider index.

2

Consumer Sends Intent

The user opens their PAE and selects "Morning Warrior" profile. The PAE constructs a consumer intent payload with fitness/motivation-heavy weights, rage-bait exclusions, and a 15-minute target. It sends this to Good Vibes via the SEP query endpoint.

3

Provider Returns Payload

Good Vibes queries its index, matches against the intent weights and filters, ranks results by relevance, and suggests a session arc. It returns 7 items with a suggested sequence: humor opener, fitness/stoicism builders, motivation peak, stretch/music closer.

4

PAE Composes Session

The consumer's PAE receives the payload and applies local preferences. It might reorder items, swap the stoicism piece for another builder, or adjust the arc based on the user's historical preferences. The PAE makes the final sequencing decision.

5

User Experiences Session

The consumer shell renders the session arc. Content is lazy-loaded from YouTube when the user reaches each item. The user watches, skips, pauses, likes. The session has a defined end — when it's done, it's done. No infinite scroll.

6

Feedback Loop (Optional)

If the user opted in, the PAE sends aggregated telemetry back to Good Vibes. The provider learns that humor openers work, stoicism didn't land in this context, and Goggins motivation was the session highlight. The PAE also updates local preferences — maybe reducing stoicism weight for morning sessions.
